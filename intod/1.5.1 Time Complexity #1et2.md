## 1.5.1 Time Complexity #1



### Core Concept

In this video, the focus shifts from general frequency counting to analyzing specific **code snippets** to determine their Time Complexity ($T(n)$). The goal is to understand how different loop increments and decrements affect the number of executions.

### Key Snippets Analyzed

* **Linear Increment:** A loop structured as `for(i=0; i<n; i++)`. Since $i$ increases by 1 each time, the statement inside executes exactly $n$ times.
* **Complexity:** $O(n)$


* **Linear Decrement:** A loop structured as `for(i=n; i>0; i--)`. Similarly, $i$ decreases by 1 until it hits 0.
* **Complexity:** $O(n)$


* **Nested Loops (Independent):** Two nested loops where both depend on $n$ (e.g., `i` from $0$ to $n$ and `j` from $0$ to $n$). The inner statement executes $n \times n$ times.
* **Complexity:** $O(n^2)$


* **Nested Loops (Dependent):** A nested loop where the inner loop limit depends on the outer loop variable (e.g., `j < i`). This results in a series sum: $1 + 2 + 3 + ... + n$, which equals $\frac{n(n+1)}{2}$.
* **Complexity:** $O(n^2)$ (since $n^2$ is the highest degree).

## 1.5.2 Time Complexity Example #2



### Core Concept

This video introduces more complex loop patterns, specifically focusing on cases where the loop variable does not change linearly (increments of 1), but rather **geometrically** or via **square roots**.

### Key Patterns Analyzed

* **Square Root Pattern:** A loop like `for(i=1; p<=n; i++)` where `p = p + i`. Here, $p$ grows as the sum of the first $k$ integers ($\frac{k(k+1)}{2}$). To find when the loop stops, we set $\frac{k^2}{2} \approx n$, leading to $k \approx \sqrt{n}$.
* **Complexity:** $O(\sqrt{n})$


* **Multiplicative/Geometric Growth:** A loop where `i` doubles each time (`i = i * 2`). If $i$ starts at 1, it follows the sequence $2^0, 2^1, 2^2, ..., 2^k$. The loop stops when $2^k > n$. Taking the log of both sides gives $k = \log_2 n$.
* **Complexity:** $O(\log n)$


* **Divisive/Geometric Decay:** A loop where `i` is halved each time (`i = i / 2`). This is the inverse of doubling and follows the same logarithmic logic.
* **Complexity:** $O(\log n)$



> **Important Takeaway:** If the loop variable grows/shrinks by a constant factor (multiplication or division), the complexity involves **$\log n$**. If it grows quadratically relative to the iterations, the complexity involves **$\sqrt{n}$**.
